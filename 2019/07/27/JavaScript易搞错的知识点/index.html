JavaScript易搞错的知识点

2019-07-27

基础类型Primitive

Primitive的表示基础类型，在JS中指非对象类型并且没有任何方法的数据，包括：string, number, bigint, boolean, null, undefined, symbol七种。例如在Vue源码中，判断一个值是否是基础类型：

function isPrimitive(value) {
  return (
    typeof value === 'string' || 
    typeof value === 'number' || 
    typeof value === 'symbol' ||
    typeof value === 'boolean'
  )
}

基础类型是不可被改变，这里的不可被改变是指如果一个变量被赋值为基础类型的数据，那么变量被改变时，会被赋予一个新值，而不会改变原来的值。

Plain JavaScript Object

Plain JavaScript Object是JS中的最简单的对象，通过字面量{}的形式、或者new Object()形式定义的对象。例如在Vue源码中，用于判断一个对象是否是一个简单对象的办法：

function isPlainObject(obj) {
  return _toString.call(obj) === '[Object Object]'
}

事件委托

事件委托利用事件冒泡，只指定一个事件处理程序，就能管理某一类型的所有事件。事件委托可以解决“事件处理程序过多”的问题，不必为每次元素添加事件，只在更高层级的DOM上添加一个事件处理程序，降低内存，提升性能。

<ul id="speak">
  <li id="speakChinese">Speak Chinese</li>
  <li id="speakEnglish">Speak English</li>
  <li id="speakFrench">Speak French</li>
</ul>

var list = document.getElementById("speak");
document.onclick = function(event) {
  event = event || window.event;
  var target = event.target || event.srcElement;
  if(target.id === "speakChinese") {
    alert("Speak Chinese")
  } else if(target.id === "speakEnglish") {
    alert("Speak English")
  } else if (target.id === "speakFrench") {
    alert("Speak French")
  }
}

JavaScript的垃圾回收

JavaScript有自动垃圾收集机制，也就是说，执行环境会负责管理代码执行过程中使用的内存。垃圾收集机制原理：找到不再使用的变量，然后释放其占用的内存。垃圾收集器按照固定的时间间隔，周期性的执行这一操作。表示无用变量的策略分为：

  1. 标记清除（mark-and-sweep）。当变量进入环境时，标记变量为“进入环境”；当变量离开坏境时，则将其标记为“离开环境”。

2008年为止，IE、Firefox、Opera、Chrome和Safari都是标记清除式垃圾收集策略，只是垃圾收集的时间间隔互有不同。

  1. 引用计数（reference counting）。跟踪每个值被引用的次数，当这个值的引用次数为0时，将其占用的内存空间回收过来。

引用计数存在严重问题：循环引用。解决策略：在不使用变量时，手工断开连接。

内存泄漏

定义：当不再用到的内存，没有及时释放，就叫内存泄漏（memory leak）

执行环境

执行环境定义了变量或函数有权访问的其他数据，每个执行环境都有一个与之关联的变量对象，环境中定义的所有对象和函数都保存在这个对象之中，用于解析器在处理数据时在后台使用。

全局执行环境是最外围的执行环境，在Web浏览器中，全局执行环境被认为是window对象。

new运算符

创建用户定义对象类型的实例或具有构造函数的内置对象的实例。new运算符执行以下操作：

  1. 创建一个空对象，即{}；
  2. 设置该对象的构造函数；
  3. 将步骤1创建的对象作为this的上下文；
  4. 如果该函数没有返回对象，则返回this。
